# 日报 Excel 格式（田浩文-日报.xlsx）

## 推荐版式（参考 2026.06.04 第 102 行）

**所有今日工作写在 C 列**，连续编号 `1.`～`6.`，**D 列留空**，行内 **合并 C:D** 单元格。

### 语气与结构

- 每条以 **「完成」「输出」「修复」「优化」「梳理」** 等动词开头（结果导向）。
- 写清 **项目/模块名 + 动作 + 结果/价值**，避免「测试脚本」「核对配置」式流水账。
- 合计 **4～6 条**，**每条 30～60 汉字**（含标点）。
- **禁止**编造用户未提及的工作。

### 示例（标准参考）

```
1. 完成 eyun-desk 统一网关接入改造，规范 AI 服务访问路径，降低客户端配置复杂度。
2. 完成 IM WebSocket 网关化改造，实现客户端与后端通信链路统一管理。
3. 完成桌面端 WSS 自签名证书兼容方案，解决打包后 IM 无法连接问题。
4. 完成 Tauri、Vite、Nginx 三套环境通信链路梳理，保证开发与生产环境行为一致。
5. 输出 IM 模块端到端联调方案，覆盖登录、工单聊天、文件上传等核心业务场景。
```

### 反例（避免，如 2026.06.05 测试行）

- 上午/下午拆成两列、条目过碎、缺少项目名与结果描述。
- 「执行某某脚本」「dry-run 通过」不宜作为正式日报条目。

## 表头（固定）

| 列 | 字段名 |
|----|--------|
| A | 日期 `YYYY.MM.DD` |
| B | 星期 |
| C | 每日工作汇报(上午）— **unified 模式下写全部条目** |
| D | 每日工作汇报(下午）— **unified 模式下留空** |
| E | 明日工作安排 |
| F | 工作注意事项 |

## Sheet

`config.json` 中 `active_sheet` 指向当前年份（如 `2026`）。
